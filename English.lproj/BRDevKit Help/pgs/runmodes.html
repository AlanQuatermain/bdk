Test Harness Run Modes

The test harness provides two different modes of operation.

The first is Apple TV Interface mode, where it displays the Apple TV's main menu, and operates in a similar manner to the real Finder application on the Apple TV. However, this requires some extra work to set up.

You will need to copy three folders from inside the Finder application on your Apple TV:

  • Contents/Patches
  • Contents/PlugIns
  • Contents/Screen Savers

Once this is done, select the option Start the AppleTV Interface in the Options window.

Click Start to launch the Apple TV Interface.

The second mode allows you to run only your own plugin, without requiring it to be copied into the application bundle, as in Apple TV Interface mode.

Select 'Run a specific plugin' in the Options window.

In the text field, enter the path to your plugin, or alternatively place the text entry cursor in the field and then drag the item from the Finder onto the text field to fill in the path automatically.

Click Start to launch your appliance in the BackRow window.

There is one other option in the Run Mode section of the Options window, which is only applicable when running in Apple TV Interface mode: You can choose whether the main menu will be scrollable like most menus, or whether it will be static, as it is on a stock Apple TV system. This option can only be changed while the test harness is not running however, since it must be set before the main menu is created.

A note on running a specific plugin:

It is important, when running your plugin from outside the test harness application bundle, that a copy of your plugin does not already exist inside it. The harness loads any plugins it finds at startup, because (for instance) a number of the Apple plugins initialise some media providrs upon which your plugin might depend. If an earlier copy of your plugin resides within the application bundle, you will likely see that rather than your updated development version when testing, which can lead to unexpected results.
